# alpha-forge 优化建议 — 量化模型与方法 + 报告输出（2026-06-25）

基于：通读现版全部代码 + 2025-2026 一手文献调研（arXiv / HuggingFace 论文库）。
本轮已**落地**的：3 个修复（HTML 数据字段消毒、买卖点阶梯除零、signals 重方法可选）
和**策略方法介绍**功能；其余为**建议**，按 ROI/工作量分级。

---

## 0. 核心判断（先看这条）

2025-2026 的硬证据高度一致：**在单名收益预测上，ML/LLM 很难稳定跑赢买入持有**，
而且失效方向危险——LLM agent 在下跌段尤其差（偏多性格），回测 edge 在知识截止日后蒸发（泄漏）。

- **FINSABER**（2505.07078）：20 年、100+ 标的，之前 LLM/择时的优势在更宽截面/更长周期下显著退化，多归因于幸存者偏差与数据窥探。
- **StockBench**（2510.02209）：GPT-5/Claude-4/Qwen3 等多数**跑不赢等权买入持有**，且**下跌段全员落后**。
- **Profit Mirage**（2510.07920）：LLM 回测收益**一旦越过知识截止日就蒸发**（预训练泄漏）。
- 高波动序列上，**没有方法能稳定击败"明天=今天"的朴素预测**（Applied Intelligence 2025）。

> **结论**：对一个**轻量、诚实**的 skill，可防守、有出样本证据的增益是
> **① 集成 ② 成本/换手控制 ③ 校准的不确定性→仓位 ④ 更强的过拟合控制 ⑤ regime 条件化**——
> **不是更大的预测器**。本 skill 现有的 deflated-Sharpe / PSR / PBO + walk-forward + 买入持有基准
> 这套"诚实机器"才是真正的护城河，应继续放在核心。下面所有建议都服从这个判断。

## 2. A. 量化模型与方法优化（按优先级）

每条标注：做什么 · 为什么/证据 · 落到哪个模块 · 工作量(S/M/L) · 算力(全 CPU 除非注明)。

### P0 — 最高 ROI、低工作量、强证据、纯 CPU

**A1. CPCV 组合净化交叉验证（替代单一 holdout 尾段）** · 工作量 M
做什么：在 `validation.py` 加 `cpcv()`——把时间切成 N 块，做"组合式"净化+禁区(purge+embargo)的
多路 train/test，产出**一整条 OOS 夏普分布 + 原生 PBO**，而非一个点估计。把 autoresearch 的
冠军评分从"单尾段"升级为 CPCV。
为什么：López de Prado；2024-25 受控实验证明 **CPCV 给出最低 PBO、最佳 DSR**（优于 K-fold/walk-forward）。
这是本清单里**证据最硬、对"诚实"主张增益最大**的一项——把"一个出样本数字"变成"分布+过拟合概率"。
模块：`validation.py`（新 `cpcv`/`pbo`）、`autoresearch.py`（冠军用 CPCV 复评）。

**A2. 换手/成本惩罚目标** · 工作量 S
做什么：在 autoresearch 评分、`multi_factor`/`models` 的权重构建里加 **L1 换手 + 成本项**，
让信号不被交易成本吃掉。
为什么：turnover-regularized 目标（2407.21791）在 10-50bps 成本下**仍能保住表现**；纯惩罚项改动极小。
模块：`optimize.py` / `autoresearch.py` / `models.py`。

**A3. 集成升级：标准化 rank-average + 时变权重** · 工作量 S-M
做什么：(a) `ensemble_top_k` 改成对标准化信号做 **rank-average** 混合；(b) 可选**时变权重**——
按近端表现衰减地给成员加权（而非等权）；(c) `models` 里把 ridge/lgbm/MLP 做 **out-of-fold stacking**。
为什么：横截面研究反复显示**简单组合预测稳定优于单个复杂模型**；AlphaForge（2406.18394）的"动态组合器"
正是把这思路做成时变权重——而它的组合器部分是 CPU 轻量的。
模块：`autoresearch.py`、`models.py`。

**A4. stepwise-SPA / White Reality Check（对"策略选择"做窥探校正）** · 工作量 M
做什么：bandit 搜过 K 个策略族后，对它们的 OOS 收益跑 **Hansen SPA / 逐步 SPA** 的 bootstrap 检验，
证明被选中的那个不是数据窥探的产物。
为什么：deflated Sharpe 是**对单条策略**的多重检验校正；SPA 是**对"从一堆里选最好"这件事**的正确检验——
两者互补。这是给"策略测试选择"环节加的一道硬关卡。
模块：`validation.py`（新 `spa_test` / `reality_check`）。

### P1 — 好 ROI、中等工作量、CPU

**A5. 共形预测区间 → 仓位** · 工作量 M
做什么：用**自适应共形(ACI/CQR / 非交换 conformal risk control)** 包住 ML 前向收益预测，得到分布无关的
预测区间；**区间越宽、仓位越小**（接到 `sizing` 的 vol-target/ERC）。
为什么：CPPS（2410.16333）等；金融非平稳，自适应/非交换变体（2310.01262 等）正合适。
模型无关、CPU、直接改善**风险控制**——比改进点预测更值。
模块：新 `conformal.py` + `sizing.py` 钩子。

**A6. regime 条件化集成** · 工作量 M
做什么：让集成/因子权重**按 regime 切换**（先用 VIX/已实现波动分箱——比全 HMM 便宜、几乎一样好）。
为什么：StockBench 记录的**下跌段全员失效**正是这里的痛点；ensemble-HMM/VIX 分箱把"对收益和协方差都按
regime 条件化"，提高 Sharpe/Calmar。直接修复熊市失效模式。
模块：`regime.py` + `autoresearch.py`。

**A7. TabM 替换单 MLP** · 工作量 S（可选依赖）
做什么：因子→收益层的 `MLPModel` 升级/并列一个 **TabM**（参数高效的 MLP 集成）。
为什么：TabM（2410.24210, ICLR2025）以接近单 MLP 的成本拿到**集成级 IC 稳定性**；CPU 可跑。
（短历史/新池子时再加 TabPFN-v2，2502.02527。）
模块：`models.py`。

**A8. 因子质量预筛 + AST 正则（抗 alpha 衰减）** · 工作量 M
做什么：(a) 在跑昂贵回测**之前**用 **AlphaEval** 式无回测打分（IC 稳定性/扰动鲁棒/多样性/金融逻辑）先杀掉脆弱因子；
(b) 在 `factor_lab` 加 **AST 距离新颖性门 + AST 复杂度惩罚**。
为什么：AlphaEval（2508.13174）+ AlphaAgent（2502.16789，已 HF 核验）——两者都纯 CPU，直接打击
**alpha 衰减与过度工程**，与现有因果校验互补。
模块：`factor_lab.py`。

**A9. 概率化 regime 断点：BOCPD + ensemble-HMM 投票** · 工作量 M
做什么：把 CUSUM/GMM 升级为**贝叶斯在线断点(BOCPD)**——输出一个**断点概率**用于门控敞口；
HMM 改为树+HMM**投票**降低单模型 regime 抖动。
为什么：BOCPD（Adams-MacKay；2025 港股实证）+ ensemble-HMM（AIMS 2025）。轻量、减少 regime whipsaw。
模块：`regime.py`。

### P2 — 研究级 / "外部训练→喂分数"（**不进**轻量主路径）

**A10. TSFM 作为离线特征**：TabPFN-TS（2501.02945，免训练）/ TimesFM / Chronos 离线产出每名前向收益
点+区间，经 `models.load_external_scores` 当一个因子喂入。**务必先重校准区间**（TSFM 校准差，2510.16060）。
**A11. GNN/关系型 alpha**（FinMamba 2502.06707、MaGNet 2511.00085）：能力缺口最大但最不轻量（要 GPU + 关系数据）。
**轻量折中**：在横截面层加**行业中性化 / 相关性聚类收缩**，便宜地拿到大部分"关系"收益。
**A12. 深度 RL 执行/组合**（2509.14385、2510.19950）：GPU 重、有 reward-hacking 风险。**只借思路**
（把 vol/尾部惩罚按 regime 条件化进 sizing 的目标），**不引神经 RL**。

### 明确"不做"（轻量场景下成本 > 收益）
多智能体辩论式因子矿工（N× API 成本换边际收益）、完整 MCTS/代码进化 agent 循环、神经 RL 执行。
**借它们的思想（约束奖励、分层提示、因子即代码），不借它们的算力。**

## 3. B. 报告输出模块优化

读完 `html_report.py`（已很成熟）后，建议的增量——核心是**把"诚实机器"变成读者看得见的东西**：

1. **策略/方法介绍**（✅ 本轮已实现，见 §4）。
2. **稳健性体检区块**（接 A1/A4）：画出 **OOS 夏普分布(箱线/小提琴) + PBO + deflated Sharpe + SPA p 值**——
   把现有的过拟合控制从"日志里的数字"变成报告里的一块图。这是最能体现 skill 差异化的一块。
3. **强制下跌切片**（StockBench 教训）：每个回测段自动附一行**"熊市/最差季度 vs 买入持有"**，
   避免只展示顺风段。
4. **成本敏感度曲线**（接 A2）：净值在 0 / 10 / 30bps 三档下并排，让读者一眼看出 edge 是否扛得住成本。
5. **新闻时效/泄漏标注**（Profit Mirage 教训）：LLM/新闻打分的信号旁标注**"仅用截止日后数据评估"**+ 来源日期
   （`newsfeed` 已逐条留日期，surface 一行"数据时效"即可）。
6. **不确定性带**（接 A5）：有共形区间时，在买卖点阶梯/预测处画出前向收益区间。
7. **顶部一句话 TL;DR**：在综合立场之上再加一行**电梯结论**（号召力 + 可分享性）。
8. **数据字段消毒**（✅ 本轮已实现，`safeHtml`）。

## 4. C. 本轮已落地

**3 个修复**（全部 Node/pytest 验证）：
- `html_report.el` 的 `html:` 走 `safeHtml` 黑名单消毒（剥离 script/img/iframe/on*=/javascript:），
  SVG 图表改走 `raw:`——数据字段（新闻/AI 文本）注入与破版被堵，良性 `<b>/<span class=pos>` 保留。
- 买卖点阶梯 `span=(hi-lo)||1` 守卫——退化(止损=现价=目标)不再 NaN 错位。
- `signals.all_methods(heavy=False)` 跳过两个重方法(walk-forward/autoresearch)，多名对照表提速 ~15×。

**策略方法介绍**（用户需求："策略测试选择部分带上每个策略方法的介绍"）：
- `param_grids.STRATEGY_INFO`：7+1 个策略族的一句话原理+适用场景（单一事实源，键与 REGISTRY 对齐）。
- `signals.METHOD_INFO` + `signals.methods_report(...)`：「信号多法对照」每行带 `desc` 介绍；
  多名自动 `heavy=False` 提速并注明跳过了哪些列。
- `autoresearch.strategy_glossary(report)`：按研究排行榜涵盖的策略族产出 `{name, intro, edge}`。
- `html_report.researchSection`：「策略选择」排行下方新增 **📖 策略方法说明** 区块。
- 新增 3 个回归测试（共 **82 passed**）。

> 报告契约新增字段：`methods.rows[].desc`（每个判定方法的介绍）、`research.glossary`
> （`[{family,name,intro,edge}]`，策略族说明）。两者"给了就渲染、不给跳过"。

## 6. 一句话总结

这套 skill 的方向是对的——**诚实优先、轻量、可离线**。2025-2026 的证据进一步确认：
别去追更大的预测器，把**集成、成本/换手、校准不确定性、CPCV/SPA 过拟合控制、regime 条件化**做扎实，
并把这套"诚实机器"在报告里**可视化**出来，就是当前性价比最高的进化路径。
所有"重模型"（TSFM/GNN/RL）一律走**外部训练→喂分数**，绝不进轻量主路径，也绝不放松 deflated-Sharpe/PBO 这道关。
